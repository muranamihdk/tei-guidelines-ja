<dataRef>

<dataRef> identifies the datatype of an attribute value, either by referencing an item in an externally defined datatype library, or by pointing to a TEI-defined data specification [22.5.3.1 Datatypes]
モジュールtagdocs — Documentation Elements
属性att.global (@xml:id, @n, @xml:lang, @xml:base, @xml:space) (att.global.rendition (@rend, @style, @rendition)) (att.global.linking (@corresp, @synch, @sameAs, @copyOf, @next, @prev, @exclude, @select)) (att.global.analytic (@ana)) (att.global.facs (@facs)) (att.global.change (@change)) (att.global.responsibility (@cert, @resp)) (att.global.source (@source))
key key¶the identifier used for this datatype specification
状態 任意
データ型 teidata.xmlName
name name¶the name of a datatype in the list provided by XML Schemas: Part 2: Datatypes
状態 任意
データ型 teidata.xmlName
ref ref¶a pointer to a datatype defined in some datatype library
状態 任意
データ型 teidata.pointer
restriction restriction¶supplies a string representing a regular expression providing additional constraints on the strings used to represent values of this datatype
状態 任意
データ型 teidata.pattern

tagdocs: dataFacet
解説

Only one of the attributes key, name, and ref may be used on any given instance of dataRef.

Neither a restriction attribute not a dataFacet child element may be used unless the dataRef refers to a datatype from the specification XML Schemas: Part 2: Datatypes with a name attribute.

例
<schemaSpec ident="myTEI"
 source="http://www.tei-c.org/Vault/P5/current/xml/tei/odd/p5subset.xml">
<!-- ... -->
 <dataRef key="teidata.enumerated"/>
<!-- ... -->
</schemaSpec>

Schematron

<sch:rule context="tei:dataRef[tei:dataFacet]">
<sch:assert test="@name" role="nonfatal">Data facets can only be specified for references to datatypes specified by
XML Schemas: Part 2: Datatypes — that is, for there to be a 'dataFacet' child there must be a @name attribute.</sch:assert>
<sch:report test="@restriction"
 role="nonfatal">Data facets and restrictions cannot both be expressed on the same data reference — that is, the @restriction attribute cannot be used when a 'dataFacet' element is present.</sch:report>
</sch:rule>
